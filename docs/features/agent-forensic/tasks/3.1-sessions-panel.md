# 3.1: Sessions Panel Model

## Description

Implement the Bubble Tea model for the Sessions Panel (left panel, 25% width). Displays all historical sessions with date, tool count, and duration. Supports keyboard navigation (j/k), session selection (Enter), search filtering (/), and focus management.

## Reference Files

- `ui/ui-design.md` — Component: Sessions Panel (UF-1) — layout, states, interactions, data binding
- `ui/prototype/main-tui.html` — interactive prototype for main TUI layout
- `design/tech-design.md` — Interface: JSONL Parser (data source)
- `design/tech-design.md` — Bubble Tea Model Test Pattern

## User Stories

- Story 1 (浏览会话调用树): AC1 — left panel loads all sessions; AC2 — selecting session triggers call tree load
- Story 4 (搜索和筛选会话): AC1 — search filters in 500ms; AC2 — date format recognition; AC3 — empty state for no results
- Story 8 (边界条件): AC5 — dir not found shows error

## Implementation Notes

- Use Bubble Tea's `tea.KeyMsg` for keyboard handling
- Store sessions as `[]Session` with filtered view for search
- Cursor position as `int`, clamped to [0, len(filtered)-1]
- Search mode: `bool` flag + `string` input buffer
- Panel width from `tea.WindowSizeMsg`
- Emit custom message on session selection (consumed by App Model)
- Virtual scrolling for large session lists
