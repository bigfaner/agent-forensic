# 2.3: Sensitive Content Sanitizer

## Description

Implement sensitive content detection and masking. Scans content for patterns matching API keys, secrets, tokens, and passwords. Replaces matched values with `***`. Returns the sanitized string and a boolean indicating whether masking occurred.

## Reference Files

- `design/tech-design.md` — Interface: Sensitive Content Sanitizer (Sanitize)
- `prd/prd-spec.md` — Data Requirements (敏感内容处理)

## Acceptance Criteria

- [ ] `internal/sanitizer/sanitizer.go` implements `Sanitize(content string) (string, bool)`
- [ ] Pattern: `(?i)(api_key|secret|token|password)[\s:=]+["']?(\S+)` matches and replaces captured value
- [ ] Sanitized string does NOT contain the original sensitive value
- [ ] Handles CJK content correctly (no false positives on Chinese characters)
- [ ] Unit tests: pattern matching, false positives, CJK content, edge cases
- [ ] Test coverage >= 95%

## User Stories

- Story 3 (查看工具调用详情): AC2 — 敏感内容脱敏
- Story 8 (边界条件): related to security requirements (运行前后SHA256不变)

## Implementation Notes

- Use `regexp.MustCompile` for pattern compilation (compile once)
- Replace only the value group (group 2), keep the key name visible
- The boolean return indicates whether any masking occurred (drives UI warning display)
- No network calls, no external lookups — pure regex matching
