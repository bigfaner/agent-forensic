# 3.5: Diagnosis Modal Model

## Description

Implement the Bubble Tea model for the Diagnosis Summary modal (centered overlay, 80% x 60%). Displays all anomaly evidence for the current session with anomaly type, tool name, duration, JSONL line number, call chain context, and thinking fragments. Supports evidence selection and jump-back to call tree.

## Reference Files

- `ui/ui-design.md` — Component: Diagnosis Summary (UF-5) — layout, states, interactions, data binding
- `ui/prototype/diagnosis.html` — interactive prototype for diagnosis modal
- `design/tech-design.md` — Interface: Diagnosis Modal (DiagnosisModal)

## User Stories

- Story 2 (定位异常操作): AC3 — d key shows diagnosis with all anomalies, line numbers, call chains

## Implementation Notes

- Load anomalies from current session's parsed data (pre-computed by detector)
- Call chain: " → " joined path from root Turn to the anomaly node
- Auto-expand parent Turn on jump-back (set `IsExpanded = true`)
- Scroll position tracked as `int` index into anomalies slice
- Modal rendered as overlay using lipgloss `Place()` for centering
- Thinking content truncated at 200 chars with "..." suffix
