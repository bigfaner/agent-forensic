# 2.2: Anomaly Detector

## Description

Implement rule-based anomaly detection that checks tool calls against threshold rules. Detects two anomaly types: slow calls (duration >= 30s) and unauthorized access (file paths outside project directory). Project directory is determined via `git rev-parse --show-toplevel` with cwd fallback.

## Reference Files

- `design/tech-design.md` — Interface: Anomaly Detector (DetectAnomalies)
- `design/tech-design.md` — Data Models (Anomaly, AnomalyType)
- `prd/prd-spec.md` — 异常标记 section (阈值定义、项目目录边界定义)

## Acceptance Criteria

- [ ] `internal/detector/anomaly.go` implements `DetectAnomalies(entries []TurnEntry, projectDir string) []Anomaly`
- [ ] Duration >= 30s (inclusive) → AnomalySlow with yellow marker
- [ ] File path outside projectDir (absolute path comparison after normalization) → AnomalyUnauthorized
- [ ] Boundary test: exactly 30.000s → marked slow (>=30s inclusive)
- [ ] Project dir determined by `git rev-parse --show-toplevel`; fallback to cwd if not in git repo
- [ ] Paths normalized to absolute before comparison
- [ ] Each Anomaly includes LineNum, ToolName, Duration, FilePath, Context (parent call chain)
- [ ] Unit test coverage >= 95%

## User Stories

- Story 2 (定位异常操作): AC1 — 耗时>=30s标黄; AC2 — 项目外路径标红
- Story 5 (回放历史会话): AC3 — 耗时>=30s高亮
- Story 8 (边界条件): AC1 — 恰好30秒标黄

## Implementation Notes

- Project directory detection runs once at startup, passed as parameter
- Path normalization: `filepath.Abs()` + `filepath.Clean()` before comparison
- Use `strings.HasPrefix(normalizedPath, normalizedProjectDir)` for boundary check
- Context chain built by tracking parent Turn during iteration
- Threshold constant: `const SlowThreshold = 30 * time.Second`
