# 1.1: Data Types & Error Definitions

## Description

Define all core data structures and error types for the agent-forensic TUI. This includes parsed JSONL structures (Session, Turn, TurnEntry), anomaly types (Anomaly, AnomalyType), stats structures (SessionStats, ToolCallSummary), and all error types (DirNotFoundError, DirPermissionError, ParseError, FileReadError, FileEmptyError, CorruptSessionError).

These types form the foundation shared by all service-layer and UI-layer components.

## Reference Files

- `design/tech-design.md` — Data Models section (Session, Turn, TurnEntry, Anomaly, SessionStats)
- `design/tech-design.md` — Error Handling section (all error types)

## Acceptance Criteria

- [ ] `internal/parser/types.go` defines Session, Turn, TurnEntry, EntryType, Anomaly, AnomalyType, SessionStats, ToolCallSummary
- [ ] All error types defined in their respective packages with `Error()`, `Unwrap()` methods where applicable
- [ ] All structs match field names and types from tech-design.md Data Models section
- [ ] Unit tests for error type formatting (Error() output matches expected patterns)
- [ ] Go module initialized (`go.mod`) with all dependencies from tech-design.md Dependencies table
- [ ] `go build ./...` succeeds

## User Stories

- Story 8 (边界条件): Error types like CorruptSessionError and FileEmptyError directly support AC4 (损坏JSONL不崩溃), AC5 (目录不存在退出), AC6 (空文件空状态)

## Implementation Notes

- Initialize Go module: `go mod init github.com/user/agent-forensic`
- Place shared types in `internal/parser/types.go`
- Place error types alongside their primary consumer (e.g., ParseError in `internal/parser/errors.go`, file-related errors in their respective packages)
- EntryType should be an iota enum: tool_use, tool_result, thinking, message
- AnomalyType should be an iota enum: AnomalySlow, AnomalyUnauthorized
- All duration fields use `time.Duration`
- ExitCode in TurnEntry is `*int` (nil for non-Bash tools)
