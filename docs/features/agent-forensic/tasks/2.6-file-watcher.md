# 2.6: File Watcher

## Description

Implement the file system watcher that monitors JSONL files for changes and emits events. Uses fsnotify with polling fallback for platforms without inotify. Detects new line appends for real-time monitoring.

## Reference Files

- `design/tech-design.md` — Interface: File Watcher (Watcher, WatchEvent)
- `design/tech-design.md` — Dependencies (fsnotify)
- `design/tech-design.md` — Watcher Integration Test Pattern

## Acceptance Criteria

- [ ] `internal/watcher/watcher.go` implements the `Watcher` interface with `Start()`, `Stop()`, `Events() <-chan WatchEvent`
- [ ] WatchEvent carries FilePath, Offset, and raw new Lines
- [ ] Detects file appends within 2 seconds
- [ ] `Stop()` cleanly closes the watcher and channel
- [ ] Integration tests using `t.TempDir()` for isolated fixture files
- [ ] 3-second timeout on test waits to prevent hangs
- [ ] Test coverage >= 80%

## User Stories

- Story 6 (实时监听当前会话): AC1 — JSONL写入后2秒内显示新节点

## Implementation Notes

- Use `github.com/fsnotify/fsnotify` for OS-native file change events
- Track file sizes to detect appends (compare pre/post size)
- Read only newly appended bytes (seek to last known offset)
- Graceful shutdown: close channel on Stop(), drain pending events
- Consider polling fallback: check file size every 1s if fsnotify unavailable
