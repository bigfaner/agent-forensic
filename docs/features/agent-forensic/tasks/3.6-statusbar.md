# 3.6: Status Bar Model

## Description

Implement the Bubble Tea model for the Status Bar (full width, 1 line, fixed at bottom). Renders key hints based on current view/mode, monitoring status indicator, and language indicator. Implements responsive truncation based on terminal width.

## Reference Files

- `ui/ui-design.md` — Component: Status Bar (UF-6) — layout, states, interactions, data binding, responsive truncation strategy
- `ui/prototype/main-tui.html` — interactive prototype showing status bar
- `design/tech-design.md` — Interface: Status Bar (StatusBarModel)

## Acceptance Criteria

- [ ] `internal/model/statusbar.go` implements `StatusBarModel` with `Init()`, `Update(msg tea.Msg)`, `View() string`
- [ ] States: Normal, Search Active, Diagnosis Active, Dashboard Active, Error
- [ ] Normal state: "j/k:nav Enter:expand Tab:detail /:search n/p:replay d:diag s:stats m:mon 监听:{status} q:quit"
- [ ] Responsive truncation: priority-1 (>=60 cols), priority-2 (>=80 cols), priority-3 (>=100 cols)
- [ ] Monitoring indicator: "监听:开" (green) / "监听:关" (text-secondary)
- [ ] Language indicator: "中" / "EN" at far right
- [ ] Mode-specific hints: search mode, diagnosis mode, dashboard mode, error mode
- [ ] Status bar updates on mode change, monitoring toggle, language switch
- [ ] Test coverage >= 85%

## User Stories

- Story 6 (实时监听当前会话): monitoring indicator shows watch status
- i18n: language indicator at far right

## Implementation Notes

- Key hints string built dynamically based on current mode + terminal width
- Width received via `tea.WindowSizeMsg`
- Monitoring status and language passed from App Model via `Update()` method
- All display text externalized through i18n `T()` function
- One-line fixed height, rendered as the last line in the layout
- Responsive truncation: silently omit lower-priority keys beyond current width
