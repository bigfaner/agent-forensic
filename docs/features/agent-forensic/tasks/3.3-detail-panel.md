# 3.3: Detail Panel Model

## Description

Implement the Bubble Tea model for the Detail Panel (bottom panel, 75% width, lower 33%). Displays full tool parameters, stdout/stderr, and thinking fragments for the selected call tree node. Supports content truncation (>200 chars), expansion (Enter), and automatic sensitive content masking.

## Reference Files

- `ui/ui-design.md` — Component: Detail Panel (UF-3) — layout, states, interactions, data binding
- `ui/prototype/main-tui.html` — interactive prototype for main TUI layout
- `design/tech-design.md` — Interface: Sensitive Content Sanitizer (data masking)

## Acceptance Criteria

- [ ] `internal/model/detail.go` implements `DetailModel` with `Init()`, `Update(msg tea.Msg)`, `View() string`
- [ ] States: Empty, Truncated, Expanded, Masked, Error
- [ ] Content >200 chars: truncated with "...truncated (Enter to expand)" notice
- [ ] Content <=200 chars (including exactly 200): displayed in full (no truncation)
- [ ] Sensitive values masked with `***`, warning banner "⚠ 内容已脱敏" shown
- [ ] Panel title: "Detail: {tool} — exit={N}, line {N}" or "Detail: {tool} — line {N}"
- [ ] Keyboard: Enter toggle truncate/expand, Tab cycle to Sessions, Esc return to Call Tree
- [ ] Virtual scroll for expanded large content
- [ ] Golden file tests for truncated, expanded, and masked rendering
- [ ] Test coverage >= 85%

## User Stories

- Story 3 (查看工具调用详情): AC1 — Tab shows full detail; AC2 — sensitive content masked; AC3 — truncate/expand toggle
- Story 8 (边界条件): AC2 — exactly 200 chars NOT truncated

## Implementation Notes

- Truncation threshold: `>200` chars (not >=200), so exactly 200 is not truncated
- Sanitize content before display (call `sanitizer.Sanitize()`)
- Store both raw and sanitized versions; show sanitized by default
- Field labels in bright cyan, content in white
- JSON pretty-print for tool_use.input
- Exit code only shown for Bash tool entries
