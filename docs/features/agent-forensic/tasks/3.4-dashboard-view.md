# 3.4: Dashboard View Model

## Description

Implement the Bubble Tea model for the Dashboard View (full-screen overlay). Displays statistics with horizontal bar charts for tool call counts and percentage bars for time distribution. Supports session switching via inline session picker overlay.

## Reference Files

- `ui/ui-design.md` — Component: Dashboard View (UF-4) — layout, states, interactions, data binding, Session Picker Overlay Layout
- `ui/prototype/dashboard.html` — interactive prototype for dashboard layout
- `design/tech-design.md` — Interface: Dashboard View (DashboardModel)
- `design/tech-design.md` — Interface: Stats Calculator (data source)

## Acceptance Criteria

- [ ] `internal/model/dashboard.go` implements `DashboardModel` with `Init()`, `Update(msg tea.Msg)`, `View() string`
- [ ] States: Loading, Populated, Refreshing, Session Picker, Error
- [ ] Bar chart rendering: `█` chars for counts, `█░` for percentages, sorted descending
- [ ] Bar scaling: longest bar = available width, others proportional
- [ ] Total duration, peak step (highlighted yellow if slow) displayed
- [ ] Session Picker: left 25% overlay with j/k navigation and Enter selection

- [ ] Golden file tests for populated and picker states
- [ ] Test coverage >= 85%

## User Stories

- Story 7 (查看统计仪表盘): AC1 — s key shows dashboard; AC2 — accurate stats; AC3 — session switch refreshes 500ms; AC4 — s/Esc returns

## Implementation Notes

- Compute stats on session change (call `CalculateStats()`)
- Bar chart width: available panel width minus label column minus count/percentage column
- `█` char for filled portion, `░` for unfilled (percentage bars)
- Session picker reuses similar rendering to Sessions Panel but as overlay
- Dashboard content outside picker dims to text-secondary when picker is active
- 500ms refresh: brief dim transition on data change
