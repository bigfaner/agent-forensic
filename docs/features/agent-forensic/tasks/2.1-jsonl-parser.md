# 2.1: JSONL Parser

## Description

Implement the JSONL stream parser that reads Claude Code session files from `~/.claude/*.jsonl`. Supports full parsing (`ParseSession`) and incremental parsing (`ParseIncremental`) for real-time updates. Handles corrupt JSON gracefully (skip line with warning), large files via streaming (first N lines), and empty files.

- `design/tech-design.md` — Interface: JSONL Parser (ParseSession, ParseIncremental)

## Acceptance Criteria

- [ ] `internal/parser/jsonl.go` implements `ParseSession(filePath string, maxLines int) (*Session, error)`
- [ ] `ParseIncremental(filePath string, lastOffset int64) ([]TurnEntry, int64, error)` implemented
- [ ] Scans `~/.claude/` directory for `*.jsonl` files
- [ ] Files > maxLines return only first maxLines entries (streaming)
- [ ] Corrupt JSON lines are skipped with warning; >50% failures escalate to CorruptSessionError
- [ ] Empty files (0 bytes) return FileEmptyError
- [ ] Unit tests: happy path (3-turn session), large file (streaming), corrupt JSONL, empty file
- [ ] Test coverage >= 90%

## Implementation Notes

- Use `bufio.Scanner` for line-by-line reading to bound memory
- `ParseSession` groups tool_use/tool_result pairs into TurnEntries with computed durations
- Turn detection: group entries by conversation turn boundaries
- Track line numbers (1-based) on each TurnEntry for evidence jumping
- Session metadata (date, tool count, duration) computed during parse
- For `ParseIncremental`, seek to `lastOffset` and read only new lines
