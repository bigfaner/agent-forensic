# Task Record: 3.5 Diagnosis Modal Model

## Summary
Implemented DiagnosisModal Bubble Tea model for the anomaly diagnosis overlay. Supports four states (No Anomalies, Has Anomalies, Evidence Selected, Error), double-line border (╔╗╚╝) for modal distinction, evidence blocks with icon + type tag (🟡 [slow] / 🔴 [unauthorized]), tool name + duration + line number, call chain context joined with →, thinking fragments truncated to 200 chars in bright cyan, reverse video highlight for selected evidence, and keyboard navigation (j/k navigate, Enter jump-back, Esc/q close). Jump-back emits JumpBackMsg with LineNum for the call tree to auto-expand parent Turn and scroll to target. Added JumpBackMsg type, DiagnosisState enum, and i18n keys (diagnosis.no_anomalies) for zh/en locales. Golden file tests for anomaly list rendering.
