# 2.5: Stats Calculator

## Description

Implement the statistics calculator that aggregates session data for dashboard display. Computes tool call counts, time distribution percentages, peak (slowest) step, and total duration.

## Reference Files

- `design/tech-design.md` — Interface: Stats Calculator (CalculateStats)
- `design/tech-design.md` — Data Models (SessionStats, ToolCallSummary)
- `ui/ui-design.md` — Component: Dashboard View (data binding)

## Acceptance Criteria

- [ ] `internal/stats/stats.go` implements `CalculateStats(session *Session) *SessionStats`
- [ ] ToolCallCounts: accurate count per tool name (0 error margin)
- [ ] ToolTimePcts: percentage (0-100) per tool, sum ≈ 100%
- [ ] PeakStep: single slowest tool call (ToolCallSummary with name + duration)
- [ ] TotalDuration computed from session
- [ ] Handles empty sessions gracefully (zero counts, nil PeakStep)
- [ ] Unit tests: count accuracy, percentage calculation, empty sessions, single-tool sessions
- [ ] Test coverage >= 90%

## User Stories

- Story 7 (查看统计仪表盘): AC2 — 统计数据准确 (工具调用计数误差0, 耗时误差<=1s)

## Implementation Notes

- Iterate over all Turns and their Entries to count tool_use messages
- Sum durations per tool for percentage calculation
- Percentage = tool total duration / grand total duration * 100
- Use `sort.Slice` for descending sort by count or duration
