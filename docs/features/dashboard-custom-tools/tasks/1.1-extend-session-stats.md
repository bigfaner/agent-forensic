# 1.1: 扩展 SessionStats 数据模型

## Description

在 `internal/parser/types.go` 中新增 `MCPServerStats` 类型，并在 `SessionStats` 结构体中添加 `SkillCounts`、`MCPServers`、`HookCounts` 三个字段。

## Reference Files

- `internal/parser/types.go` — 现有 SessionStats 定义
- `docs/features/dashboard-custom-tools/design/tech-design.md` — Interface 1: SessionStats 扩展字段

## Acceptance Criteria

- [ ] `MCPServerStats` 类型定义正确：`Total int`、`Tools map[string]int`
- [ ] `SessionStats` 新增三个字段：`SkillCounts map[string]int`、`MCPServers map[string]*MCPServerStats`、`HookCounts map[string]int`
- [ ] `go build ./...` 编译通过
- [ ] 现有测试 `go test ./internal/parser/...` 全部通过（无破坏性变更）

## User Stories

Story 1（Skill 明细）、Story 2（MCP 分组）、Story 3（Hook 次数）、Story 4（无数据不渲染）均依赖此数据模型。

## Implementation Notes

- 新字段初始化为非 nil map，与现有 `ToolCallCounts` 保持一致
- `MCPServerStats` 定义在 `types.go` 同文件，不新建文件
- 此任务为 breaking：下游 `CalculateStats` 和渲染层均依赖新字段
