# 2.3: 集成自定义工具区块到仪表盘

## Description

将 `renderCustomToolsBlock()` 集成到 `renderDashboard()` 末尾，完成 existing-page 集成。

## Reference Files

- `internal/model/dashboard.go` — renderDashboard() 集成点
- `docs/features/dashboard-custom-tools/design/tech-design.md` — Integration Specs 章节
- `docs/features/dashboard-custom-tools/ui/ui-design.md` — Component Placement 章节

## Acceptance Criteria

- [ ] `renderDashboard()` 末尾（`return b.String()` 之前）追加 `b.WriteString(m.renderCustomToolsBlock(m.width - 4))`
- [ ] 仪表盘在有自定义工具数据时正确展示「自定义工具」区块
- [ ] 仪表盘在无自定义工具数据时外观与修改前完全一致（区块不渲染）
- [ ] `go build ./...` 编译通过
- [ ] `go test ./internal/model/...` 全部通过

## User Stories

**Story 4 AC**：Given 无 Skill/MCP/Hook 数据 → 仪表盘外观与当前版本一致（区块不渲染）

**Story 1 AC**：Given session 含 Skill 调用 → 仪表盘「自定义工具」区块 Skill 列显示每个 skill 名称和次数

## Implementation Notes

- 集成点：`internal/model/dashboard.go` `renderDashboard()` 末尾
- 插入位置：`return b.String()` 之前
- 数据来源：`m.stats`（已由 `Refresh()` 填充，无需额外调用）
- session 选择器由 `View()` 通过 `renderPicker()` 单独追加，不在 `renderDashboard()` 内部，本区块天然位于选择器上方
