# 1.2: 扩展 CalculateStats 聚合逻辑

## Description

在 `internal/stats/stats.go` 中实现三个内部解析函数，并在 `CalculateStats()` 中调用它们填充 `SessionStats` 的新字段。

## Reference Files

- `internal/stats/stats.go` — 现有 CalculateStats 实现
- `internal/parser/types.go` — SessionStats、MCPServerStats 定义（1.1 完成后）
- `docs/features/dashboard-custom-tools/design/tech-design.md` — Interface 2: 三个内部函数签名

## Acceptance Criteria

- [ ] `parseSkillInput(rawInput string) string`：正常解析返回 `skill` 字段值；JSON 无 `skill` 字段或解析失败时返回 input 前 20 字符（`[]rune` 截断）
- [ ] `parseMCPToolName(toolName string) (server, tool string)`：匹配 `mcp__<server>__<tool>` 返回两段；不匹配返回 `("", "")`
- [ ] `parseHookMarker(text string) string`：包含已知标记（PreToolUse / PostToolUse / Stop / user-prompt-submit-hook）返回标记名；无匹配返回 `""`；`<user-prompt-submit-hook>` 去除尖括号后返回 `user-prompt-submit-hook`
- [ ] `CalculateStats()` 正确填充 `SkillCounts`、`MCPServers`（含 Total 和 Tools）、`HookCounts`
- [ ] 同一 turn 内同一 hook 类型出现多次，每次单独计数
- [ ] `go test ./internal/stats/...` 覆盖率 ≥ 90%，包含 Story 1–7 的 AC 场景

## User Stories

**Story 1 AC**：Given session 含 Skill 调用 forge:brainstorm×3、forge:execute-task×5 → SkillCounts["forge:brainstorm"]=3，SkillCounts["forge:execute-task"]=5，总和=8=ToolCallCounts["Skill"]

**Story 2 AC**：Given mcp__web-reader__webReader×10、mcp__web-reader__search×2、mcp__ones-mcp__addIssueComment×8 → MCPServers["web-reader"].Total=12，MCPServers["web-reader"].Tools["webReader"]=10

**Story 3 AC**：Given PostToolUse 触发 87 次 → HookCounts["PostToolUse"]=87

**Story 5 AC**：Given Skill input JSON 无 skill 字段 → SkillCounts 中以 input 前 20 字符为 key 计数

## Implementation Notes

- Hook 解析：扫描 `EntryMessage` 类型（role=user）的 `Output` 字段
- `CalculateStats` 初始化时将三个新字段设为非 nil map
- `parseHookMarker` 用 `strings.Contains` 匹配，不用正则
- MCP server 的 `Total` 字段在每次 tool 计数后同步累加
