# 2.1: 实现 renderCustomToolsBlock 渲染函数

## Description

在 `internal/model/dashboard.go` 中实现 `renderCustomToolsBlock(width int) string` 及其三个子函数 `renderSkillCol`、`renderMCPCol`、`renderHookCol`，覆盖所有状态和布局规则。

## Acceptance Criteria

- [ ] 三类均无数据时返回 `""`（区块不渲染）
- [ ] width ≥ 80 时三列并排，每列宽度 `(width - 6) / 3`，最小 18；若 `(width-6)/3 < 18` 回退单列
- [ ] width < 80 时单列堆叠，顺序 Skill → MCP → Hook，列间空一行
- [ ] 无数据的列显示 `(none)`（fg-muted color "240"）
- [ ] Skill 列：按调用次数降序，次数相同按名称字母升序；超过 10 条显示 `... +N more`
- [ ] MCP 列：server 按总次数降序；每 server 下工具按次数降序，次数相同按名称升序；超过 5 个工具显示 `... +N more`；列标题附 ` *`，区块底部注明 `* 仅统计 mcp__ 前缀工具`
- [ ] Hook 列：按次数降序，次数相同按名称升序；超过 10 条显示 `... +N more`；hook 类型去除首尾 `<>` 后展示原始标签名
- [ ] 名称超过 22 字符时截断并附 `…`（`[]rune` 截断）
- [ ] 计算中状态：数据区域显示 `计算中…`（fg-muted）
- [ ] 统计失败状态：数据区域显示 `统计失败`（fg-muted）
- [ ] `go test ./internal/model/...` 覆盖率 ≥ 80%

## User Stories

**Story 4 AC**：Given 三类均无数据 → renderCustomToolsBlock 返回 `""`，仪表盘外观与当前版本一致

**Story 6 AC**：Given server 下 8 个工具 → 展示前 5 个 + `... +3 more`，server 总次数仍为所有工具之和

**Story 7 AC**：Given width=60 → 单列堆叠，Skill → MCP → Hook 顺序，内容完整可读

## Implementation Notes

- 颜色常量：fg-primary="15"，fg-secondary="252"，fg-dim="242"，fg-accent="51"，fg-muted="240"
- 区块标题 `自定义工具` 使用 `lipgloss.NewStyle().Bold(true).Foreground("15")`
- 列标题使用 `lipgloss.NewStyle().Foreground("51")`
- 区块高度 = 最高列行数；较短列不补空行
- 此任务只实现函数，不修改 `renderDashboard()` 调用点（由 2.3 集成）
