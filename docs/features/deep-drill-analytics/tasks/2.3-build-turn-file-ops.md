# 2.3: Build Turn File Operations View (UF-3)

## Description

Build the Turn Overview file operations component (UI Function 3). When a Turn header is selected, the Detail panel's Turn Overview shows a "files:" section listing files read/edited in that turn. Create the rendering logic for file path display with truncation and R×N/E×N formatting.

## Reference Files

- `ui/ui-design.md` — Component: Turn Overview File Operations (UF-3)
- `ui/prototype/detail-fileops.html` — interactive prototype
- `design/tech-design.md` — Integration 2: Turn Overview File List → Detail Panel
- `internal/model/detail.go` — existing detail panel

## Acceptance Criteria

- [ ] "files:" section label in bright cyan (same style as "tools:" label)
- [ ] File rows: path (truncated to panel width, `...filename` format) + `R×N` (green) + `E×N` (red)
- [ ] Rows sorted by operation count descending
- [ ] Max 20 rows; overflow shows `+N more` in text-secondary
- [ ] Section hidden when turn has no Read/Write/Edit calls
- [ ] Unit tests: files section rendered with mock FileOpStats, hidden when empty

## User Stories

- **Story 4** (Turn): "Given 我在 Call Tree 中选中一个 Turn header, When Detail 面板展示 Turn Overview, Then Turn Overview 中包含该 Turn 内读写/编辑的文件列表（路径截断至面板宽度，按操作次数降序排列，最多 20 条）"
- **Story 4** (empty): "Given 选中 Turn 内无任何 Read/Write/Edit 调用, When Detail 面板展示 Turn Overview, Then 文件列表区块不显示"

## Implementation Notes

- This is the BUILD task — create the rendering function only
- Do NOT wire into `buildTurnOverview()` yet (that's the integration task 3.3)
- Path truncation: if path exceeds panel width, show `...filename.go` (keep the filename, truncate from left)
- Create a helper function like `renderFileList(fileOps *FileOpStats, width int) string`
