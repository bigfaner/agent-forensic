# 1.2: Implement File Path Extraction

## Description

Implement Interface 2 from tech-design.md: Extract file paths from Read/Write/Edit tool calls and aggregate into `FileOpStats`. Define the `FileOpStats` and `FileOpCount` data models.

## Reference Files

- `design/tech-design.md` — Interface 2: File Path Extraction
- `design/tech-design.md` — Data Model 1: FileOpStats, Data Model 2: FileOpCount
- `internal/stats/stats.go` — existing stats module

## Acceptance Criteria

- [ ] `FileOpStats` struct defined with `Files map[string]*FileOpCount`
- [ ] `FileOpCount` struct defined with `ReadCount`, `EditCount`, `TotalCount` fields
- [ ] `ExtractFilePaths(entries []TurnEntry) *FileOpStats` aggregates file operations
- [ ] Read tool calls increment `ReadCount`; Write/Edit tool calls increment `EditCount`
- [ ] Entries without `input.file_path` are silently skipped
- [ ] `TotalCount` computed as `ReadCount + EditCount`
- [ ] Unit tests: entries with file_path, entries without file_path, mixed tools, empty slice
- [ ] Coverage target: 90% for new code

## User Stories

- **Story 3** (partial): Provides the file operation data for the Dashboard ranking panel
- **Story 4** (partial): Provides the file list for Turn and SubAgent detail views

## Implementation Notes

- Tool names to check: "Read", "Write", "Edit" (case-sensitive match against TurnEntry.ToolName)
- File path source: `entry.Input` JSON field `file_path`
- Use `json.Unmarshal` on entry.Input to extract the path; skip on failure
- Path should be stored as-is; relative path conversion is a rendering concern
