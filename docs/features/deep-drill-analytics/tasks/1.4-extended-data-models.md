# 1.4: Extend SessionStats Data Models

## Description

Implement Interface 4 from tech-design.md: Extend the existing `SessionStats` struct with new fields for file operations, hook details, and SubAgent statistics. Define `SubAgentStats` model and the `visibleNode` extension for CallTree depth-2 rendering.

## Reference Files

- `design/tech-design.md` — Interface 4: Extended Session Stats
- `design/tech-design.md` — Data Model 4: SubAgentStats
- `design/tech-design.md` — Data Model 5: CallTree visibleNode Extension
- `design/tech-design.md` — Error Handling: CallTreeModel error state (subAgentErrors field)
- `internal/stats/stats.go` — existing SessionStats struct
- `internal/model/calltree.go` — existing visibleNode struct

## Acceptance Criteria

- [ ] `SessionStats` extended with `FileOps *FileOpStats`, `HookDetails []HookDetail`, `SubAgents map[string]*SubAgentStats`
- [ ] `SubAgentStats` struct defined with `ToolCounts`, `ToolDurs`, `FileOps`, `ToolCount`, `Duration`
- [ ] `visibleNode` extended with `depth int` (0=turn, 1=tool, 2=subagent child) and `subIdx int` (-1 if not subagent child)
- [ ] `CallTreeModel` extended with `subAgentErrors map[int]error`
- [ ] `CalculateStats` (or equivalent) populates `FileOps` and `HookDetails` from parsed entries
- [ ] Existing code that reads `SessionStats` still compiles (backward-compatible additions)
- [ ] Unit tests: SessionStats with new fields, SubAgentStats calculation, empty cases
- [ ] Coverage target: 90% for new code

## User Stories

- **Story 1** (partial): SubAgentStats provides data for inline expand statistics
- **Story 2** (partial): SubAgentStats provides data for overlay three-section view
- **Story 3** (partial): FileOps provides data for Dashboard file ranking
- **Story 4** (partial): FileOps per-turn/SubAgent provides data for Detail views
- **Story 5** (partial): HookDetails provides data for Dashboard Hook panel

## Implementation Notes

- This is a breaking task because it modifies the existing `SessionStats` struct (additive fields, but struct layout changes)
- All new fields are pointer/slice types so zero-value is nil (backward compatible for existing consumers)
- `SubAgents` map key is the subagent file path (as returned by ScanSubagentsDir)
- `CalculateStats` should call `ExtractFilePaths` and `ParseHookWithTarget` internally
- `visibleNode.depth` and `visibleNode.subIdx` default values: depth=1 (existing tool level), subIdx=-1 (not a subagent child)
