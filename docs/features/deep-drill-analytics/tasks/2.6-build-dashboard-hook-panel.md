# 2.6: Build Dashboard Hook Analysis Panel (UF-6)

## Description

Build the Dashboard Hook Analysis panel component (UI Function 6). Create rendering functions for two sections: Hook Statistics (grouped by HookType::Target) and Hook Timeline (by Turn). Color-coded markers by hook type.

## Reference Files

- `ui/ui-design.md` — Component: Dashboard Hook Analysis Panel (UF-6)
- `ui/prototype/dashboard-analytics.html` — interactive prototype
- `design/tech-design.md` — Integration 5: Hook Analysis Panel → Dashboard
- `internal/model/dashboard_custom_tools.go` — existing custom tools rendering

## Acceptance Criteria

- [ ] Hook Statistics section: header "Hook Statistics" in bold white, `HookType::Target ×N` rows sorted by count descending
- [ ] Target extraction failure: show only `HookType` (no `::` suffix)
- [ ] Hook Timeline section: header "Hook Timeline (by Turn)" in bold white
- [ ] Legend row: `● PreToolUse(green)  ● PostToolUse(cyan)  ● Stop(yellow)  ● user-prompt(magenta)`
- [ ] Each row: `T{N}` turn label (3 chars, right-aligned) + `●HookType::Target` markers
- [ ] Marker colors: PreToolUse=bright green, PostToolUse=bright cyan, Stop=bright yellow, user-prompt-submit=bright magenta
- [ ] Max 30 markers per line; overflow wraps with continuation indent
- [ ] Both sections hidden when no hooks (empty string)
- [ ] Unit tests: statistics grouping, timeline rendering, empty case, target fallback

## User Stories

- **Story 5** (stats): "Given 会话包含 Hook 触发记录, When 我在 Dashboard 中查看 Hook 面板, Then Hook 统计按 HookType::TargetCommand 分组显示"
- **Story 5** (timeline): "When 我查看 Hook 时序面板, Then 按 Turn 编号升序展示每种 Hook 类型的触发时间线"
- **Story 5** (empty): "Given 会话不包含任何 Hook 触发记录, When 我打开 Dashboard, Then Hook 分析面板不显示"

## Implementation Notes

- This is the BUILD task — create the rendering functions only
- Do NOT replace existing Hook rendering in dashboard_custom_tools.go yet (that's the integration task 3.5)
- Create helper functions for statistics grouping and timeline rendering
- Can be in a new file or as unexported functions in dashboard_custom_tools.go
- HookDetails are pre-computed by stats layer; this task only renders them
- Turn labels: format as `T1`, `T2`, etc. (3-char wide, right-aligned with spaces)
