# 3.1: Integrate SubAgent Expand into Call Tree (UF-1)

## Description

Wire the SubAgent Inline Expand component (built in 2.1) into the existing Call Tree panel. Modify `toggleExpand()` in `calltree.go` to trigger SubAgent lazy loading when a SubAgent node is expanded. Wire depth-2 navigation and Detail panel synchronization.

## Reference Files

- `design/tech-design.md` — Integration 1: SubAgent Inline Expand → Call Tree Panel
- `ui/ui-design.md` — Component: SubAgent Inline Expand (UF-1) Interactions table
- `internal/model/calltree.go` — target file, insertion point: `toggleExpand()` method

## Acceptance Criteria

- [ ] `toggleExpand()` detects SubAgent nodes and triggers `ScanSubagentsDir()` + `ParseSubAgent()` lazy load
- [ ] Loading state shown while parsing (async, non-blocking)
- [ ] Parsed SubAgent children injected into `TurnEntry.Children` and visibleNodes rebuilt at depth 2
- [ ] Parse errors stored in `subAgentErrors[entryIdx]`, node stays collapsed with `⚠` suffix
- [ ] `Enter` on collapsed SubAgent → expand; `Enter` on expanded SubAgent → collapse
- [ ] j/k navigation works for depth-2 child nodes (same highlight as depth-1)
- [ ] Selecting a SubAgent child updates Detail panel with SubAgent statistics view
- [ ] `Enter` on error-state node does not expand; detail shows full error message
- [ ] Component visible at correct position in Call Tree (depth 2, below SubAgent parent)
- [ ] Existing Call Tree behavior unchanged for non-SubAgent nodes

## User Stories

- **Story 1** (integration): Full SubAgent expand flow — select, Enter, see children, navigate, Detail sync
- **Story 1** (error integration): Error states display correctly in tree and detail

## Implementation Notes

- Target file: `internal/model/calltree.go`
- Insertion point: `toggleExpand()` method
- Data source: `parser.ScanSubagentsDir()` + `parser.ParseSubAgent()` → children injected into TurnEntry
- When `toggleExpand()` encounters a SubAgent node: check subAgentErrors first (if error exists, show detail message instead of expanding), then trigger async parse if not yet loaded
- Async parse sends `SubAgentLoadDoneMsg` which rebuilds visibleNodes
- Ensure j/k cursor movement correctly handles depth-2 entries in the flat visibleNodes list
