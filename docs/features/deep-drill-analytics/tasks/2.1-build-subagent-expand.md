# 2.1: Build SubAgent Inline Expand Component (UF-1)

## Description

Build the SubAgent inline expand component for the Call Tree panel (UI Function 1). When a SubAgent node is selected and Enter is pressed, render child tool calls at depth 2 (indented 2 additional spaces). Implement collapsed, loading, expanded, error, and overflow states.

## Reference Files

- `ui/ui-design.md` — Component: SubAgent Inline Expand (UF-1)
- `ui/prototype/calltree-subagent.html` — interactive prototype
- `design/tech-design.md` — Integration 1: SubAgent Inline Expand → Call Tree Panel
- `design/tech-design.md` — Interfaces — data contracts (SubAgentStats, visibleNode extension)
- `internal/model/calltree.go` — existing CallTreeModel

## Acceptance Criteria

- [ ] SubAgent children rendered at depth 2 with `├─ └─` tree connectors
- [ ] Collapsed state: `├─ SubAgent ×3 (4.8s) 📦` (ASCII: `[A]`)
- [ ] Loading state: `📦 ⏳` suffix (ASCII: `[A] ...`)
- [ ] Expanded state: children visible, navigable via j/k
- [ ] Error state: `📦 ⚠` suffix (ASCII: `[A] !`), children hidden
- [ ] Overflow: >50 children → `│  ... +N more` in text-secondary
- [ ] Children sorted by JSONL appearance order
- [ ] Error label dispatch function: `errorLabel(err)` maps error types to short labels
- [ ] Unit tests: each state renders expected substrings; mock SubAgentStats data

## User Stories

- **Story 1**: "Given 会话包含至少 1 个 SubAgent 工具调用, When 我在 Call Tree 中选中该 SubAgent 节点并按 Enter, Then Call Tree 内联展示该 subagent 的工具调用列表（>=3 层缩进）"
- **Story 1** (error): "Given SubAgent 节点的 JSONL 文件不存在或解析失败, When 按 Enter, Then 节点保持折叠状态，显示 ⚠ 标记"

## Implementation Notes

- This is the BUILD task — create the rendering logic and state management only
- Do NOT wire into `toggleExpand()` yet (that's the integration task 3.1)
- Create helper functions for rendering SubAgent children rows
- `visibleNode` with `depth=2` and `subIdx >= 0` identifies SubAgent child nodes
- Use existing lipgloss styles from the base design system
- Emoji/ASCII fallback: check `TERM` env or `--ascii` flag for `📦`/`[A]`, `⏳`/`...`, `⚠`/`!`
