# 2.5: Build Dashboard File Operations Panel (UF-5)

## Description

Build the Dashboard File Operations panel component (UI Function 5). Create `FileOpsPanel` as a stateless rendering struct that produces a horizontal bar chart of file operation statistics for the Dashboard overlay. New file: `internal/model/dashboard_fileops.go`.

## Reference Files

- `ui/ui-design.md` — Component: Dashboard File Operations Panel (UF-5)
- `ui/prototype/dashboard-analytics.html` — interactive prototype

- `design/tech-design.md` — New File Exports: `internal/model/dashboard_fileops.go`
- `design/tech-design.md` — Data binding table for UF-5

## Acceptance Criteria

- [ ] `FileOpsPanel` struct (stateless) with `NewFileOpsPanel()` constructor
- [ ] `Render(stats *FileOpStats, width int) string` — returns formatted panel or empty string if nil/no files
- [ ] Section header: "File Operations (top 20)" in bold white
- [ ] Divider: `────` in text-secondary
- [ ] Each row: path (40 chars truncated with `...` prefix) + bar (`█` proportional, max 20 chars) + `R×N` bright green + `E×N` bright red + total count
- [ ] Rows sorted by total operations descending, max 20
- [ ] Overflow: "+N more" in text-secondary at bottom
- [ ] Hidden when session has no Read/Write/Edit calls (Render returns "")
- [ ] `renderBar()` helper produces single row
- [ ] Unit tests: Render with mock stats contains expected paths/counts, Render with nil returns "", overflow display

## User Stories

- **Story 3**: "Given 会话包含至少 1 次 Read/Write/Edit 工具调用, When 我在 Dashboard 中查看文件读写面板, Then 显示水平柱状图：按文件路径聚合操作次数，显示 Read ×N / Edit ×M 计数，按总操作次数降序排列，最多展示 top 20 文件"
- **Story 3** (empty): "Given 会话不包含任何 Read/Write/Edit 工具调用, When 我打开 Dashboard, Then 文件读写面板不显示"

## Implementation Notes

- This is the BUILD task — create the component in a NEW file `internal/model/dashboard_fileops.go`
- Do NOT wire into Dashboard View() yet (that's the integration task 3.4)
- Follow the exact API spec from tech-design.md Integration 4 New File Exports section
- Not a bubbletea.Model — stateless rendering function called from dashboard View()
- Bar scaling: `len = int(float64(barWidth) * float64(total) / float64(maxCount))`
- Path truncation: if path > 40 chars, show `...` + last 37 chars
