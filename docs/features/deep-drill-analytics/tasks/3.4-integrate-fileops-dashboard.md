# 3.4: Integrate File Operations Panel into Dashboard (UF-5)

## Description

Wire the File Operations Panel (built in 2.5) into the Dashboard overlay's View() method. Insert the panel below the existing Custom Tools block. The panel renders when file operations exist and is hidden otherwise.

## Reference Files

- `design/tech-design.md` — Integration 4: File Operations Panel → Dashboard
- `ui/ui-design.md` — Component: Dashboard File Operations Panel (UF-5) Placement
- `internal/model/dashboard.go` — target file, insertion point: Dashboard View() after Custom Tools
- `internal/model/dashboard_fileops.go` — FileOpsPanel component (built in 2.5)

## Acceptance Criteria

- [ ] Dashboard View() includes FileOpsPanel output after Custom Tools block
- [ ] Panel data sourced from `SessionStats.FileOps`
- [ ] Panel hidden (not rendered) when `FileOps` is nil or has no files
- [ ] Panel visible at correct position in Dashboard (after Custom Tools)
- [ ] `Tab` in Dashboard cycles focus to File Operations section (header highlighted cyan)
- [ ] j/k scroll works within the Dashboard including the new panel

## User Stories

- **Story 3** (integration): User opens Dashboard and sees File Operations ranking panel below Custom Tools

## Implementation Notes

- Target file: `internal/model/dashboard.go`
- Insertion point: Dashboard View() method, after Custom Tools section
- Data source: `SessionStats.FileOps` passed to `FileOpsPanel.Render(stats, width)`
- The `FileOpsPanel` is stateless — just call `Render()` and append to Dashboard output
- Ensure Tab focus cycling includes the new section
