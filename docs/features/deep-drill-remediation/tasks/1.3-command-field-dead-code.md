# 1.3: Add Command field to SubAgentStats + remove SubAgentLoadMsg

## Description

Two changes:

1. **Add `Command string` field** to `parser.SubAgentStats` struct. Compute it in `stats.ComputeSubAgentStats` (or equivalent) from the first tool call: `"{ToolName}: {primary_arg}"` (e.g., `"Edit: internal/model/app.go"`).

2. **Remove `SubAgentLoadMsg` struct** from `internal/model/subagent_overlay.go` (lines 44-48). This is dead code — no code path produces this message. Its existence risks someone adding an async loading path that could get stuck.

## Reference Files

- `design/tech-design.md` Interface 5 (Command Field) + Data Models (SubAgentOverlayModel modified)
- `internal/parser/types.go:104-113` — SubAgentStats struct
- `internal/model/subagent_overlay.go:44-48` — SubAgentLoadMsg (to remove)
- `internal/stats/stats.go` — where Command computation will be added

## Acceptance Criteria

- [ ] `parser.SubAgentStats` has `Command string` field
- [ ] Command is computed from first tool call: format `"{ToolName}: {ExtractToolCommand(toolName, input)}"`
- [ ] SubAgent with zero tool calls → `Command == ""`
- [ ] SubAgentLoadMsg type does not exist: `grep -r "SubAgentLoadMsg" internal/` returns empty
- [ ] All existing tests pass: `go test ./...`
- [ ] No code references SubAgentLoadMsg: `grep -r "SubAgentLoadMsg" internal/` returns empty

## User Stories

- **Story 3: Recover from SubAgent Loading Failures** — removing SubAgentLoadMsg ensures no async loading path can produce a stuck state (AC: "the `SubAgentLoadMsg` type does not exist in the codebase")
- **Story 6: See Meaningful SubAgent Overlay Title** — Command field provides the data source for the overlay title

## Hard Rules

- Command computation MUST use `stats.ExtractToolCommand` (from task 1.2) for consistency
- SubAgentLoadMsg removal is a search-and-destroy operation — grep first, then delete
- Adding Command field to SubAgentStats is additive — existing code gets zero-value `""`

## Implementation Notes

Command derivation logic:
```go
// In computeSubAgentStats or similar:
if len(children) > 0 && children[0].Type == parser.EntryToolUse {
    cmd := stats.ExtractToolCommand(children[0].ToolName, children[0].Input)
    if cmd != "" {
        stats.Command = children[0].ToolName + ": " + cmd
    }
}
```

For `computeSubAgentStatsFromTurns`, derive from the first `EntryToolUse` entry in the turns.
