# 2.6: Add hook section scroll + meaningful overlay title

## Description

Two enhancements to the SubAgent overlay:

1. **Hook section scroll**: When hook section has >20 items, render a scrollable viewport with scrollbar. Add `hookScrollOff int` to `SubAgentOverlayModel`. Handle `↑`/`↓` within the focused hook section.

2. **Meaningful overlay title**: Display the SubAgent's `Command` field in the overlay title instead of generic label.

## Reference Files

- `design/tech-design.md` Interface 4 (Overlay Scroll State), Integration 7 (Overlay Scroll), Interface 5 (Command Field)
- `internal/model/subagent_overlay.go` — SubAgentOverlayModel struct, Update(), View(), renderTitle(), renderHookStatsSection
- `internal/parser/types.go` — SubAgentStats.Command field (added in task 1.3)
- `docs/conventions/tui-layout-ui.md` — scrollbar characters: track `"│"` (238), thumb `"┃"` (248)

## Acceptance Criteria

### Hook section scroll:
- [ ] `SubAgentOverlayModel` has `hookScrollOff int` field
- [ ] `renderHookStatsSection` accepts `scrollOff` and `maxLines` parameters
- [ ] When hook items > maxLines: scrollbar track + thumb rendered, items scrolled by `hookScrollOff`
- [ ] When hook items <= maxLines: no scrollbar, all items visible
- [ ] `↑`/`↓` in focused hook section updates `hookScrollOff`
- [ ] Scroll at top (hookScrollOff == 0): `↑` is no-op
- [ ] Scroll at bottom (hookScrollOff == maxScroll): `↓` is no-op
- [ ] Zero hook items: empty state, no crash
- [ ] One hook item: displays without scrollbar

### Overlay title:
- [ ] `renderTitle()` displays `stats.Command` when non-empty
- [ ] Title format: `"SubAgent: {Command} — {N} tools, {Xs}"` when Command available
- [ ] Title format: `"SubAgent — {N} tools, {Xs}"` when Command is empty
- [ ] Long commands truncate with `...` suffix within panel border
- [ ] Special characters (`|`, `>`, `'`) render verbatim

### Tests:
- [ ] Golden test: >20 hook items with scrollbar
- [ ] Golden test: exactly 20 items (no scrollbar)
- [ ] Golden test: overlay title with command string
- [ ] Golden test: overlay title truncation at 80 columns
- [ ] `go test ./internal/model/ -run TestOverlay -v` passes

## User Stories

- **Story 6: See Meaningful SubAgent Overlay Title**
  - Given SubAgent with tool calls → title shows `"SubAgent: Edit: app.go — 12 tools, 3.2s"`
  - Given SubAgent with zero tools → title shows `"SubAgent — 0 tools, 0.0s"`
  - Given command exceeds overlay width → truncates with `...`
  - Given command with special chars → renders verbatim

- **Story 7: Scroll Through Large Hook Lists in Overlay**
  - Given >20 hook items → scrollable viewport with scrollbar
  - Given exactly 20 items → all visible, no scrollbar
  - Given single item → no scrollbar
  - Given zero items → empty state
  - Given scrolled to bottom → `↓` is no-op
  - Given scrollable section → `↑`/`↓` scrolls within section

## Hard Rules

- `hookScrollOff` is a single int field — no additional scroll state fields
- Scrollbar uses convention characters: track `"│"` (color 238), thumb `"┃"` (color 248)
- `maxLines` comes from existing `sectionHeightsFixed()` or equivalent height allocation
- Title truncation uses shared `truncRunes()` from truncate.go

## Implementation Notes

### Hook scroll changes:
1. Add `hookScrollOff int` to `SubAgentOverlayModel`
2. In `Update()`: when `focusedSection == 1` (hooks), handle `"up"`/`"down"` to adjust `hookScrollOff`
3. `clampHookScroll()` method: ensure `0 <= hookScrollOff <= max(len(details)-maxLines, 0)`
4. Update `renderHookStatsSection(details, width, scrollOff, maxLines)` to render viewport
5. Reset `hookScrollOff = 0` when opening overlay or when hook data changes

### Title changes:
1. In `renderTitle()`: access `m.stats.Command` (from task 1.3)
2. Format: `"SubAgent"` + optional `": "+Command` + `" — N tools, X.Xs"`
3. Truncate with `truncRunes()` if total title exceeds overlay width
