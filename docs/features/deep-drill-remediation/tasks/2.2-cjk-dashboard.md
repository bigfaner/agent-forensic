# 2.2: Fix CJK rendering in Dashboard (FileOps + Tool Stats)

## Description

Replace byte-based width calculations with `runewidth.StringWidth()` in two dashboard files: `dashboard_fileops.go` (path padding) and `dashboard.go` (tool name labels).

## Reference Files

- `design/tech-design.md` Integration 2 (Dashboard FileOps) + Integration 3 (Dashboard Tool Stats)
- `internal/model/dashboard_fileops.go` — column width (lines 53-83), renderRow (lines 102-144)
- `internal/model/dashboard.go` — tool stats rendering (lines 410-504)
- `internal/model/truncate.go` — shared `truncatePathBySegment` (from task 1.1)

## Acceptance Criteria

- [ ] `dashboard_fileops.go`: all `len()` calls for path width replaced with `runewidth.StringWidth()`
- [ ] `dashboard_fileops.go`: `utf8.RuneCountInString()` for count columns kept (pure ASCII digits per convention §7)
- [ ] `dashboard_fileops.go`: path truncation uses `truncatePathBySegment()` from truncate.go
- [ ] `dashboard.go`: tool name truncation uses `runewidth.StringWidth()` and shared `truncRunes()`
- [ ] `dashboard.go`: peak step name truncation (line 418) uses `runewidth.StringWidth()`
- [ ] Golden test at 80x24 with CJK file paths + tool names passes
- [ ] `grep -n '\blen(' internal/model/dashboard_fileops.go` returns no width-related matches
- [ ] `grep -n '\blen(name)\|len(e\.name)' internal/model/dashboard.go` returns no matches
- [ ] `go test ./internal/model/ -run TestDashboard -v` passes

## User Stories

- **Story 1: View CJK File Paths Without Corruption** — Dashboard FileOps and Tool Stats panels render CJK correctly
  - Given CJK file paths in FileOps → paths align correctly in columns
  - Given CJK tool name labels in Tool Stats → labels truncate correctly at panel boundary

## Hard Rules

- `len()` MUST NOT be used for any visible width calculation
- Add `"github.com/mattn/go-runewidth"` import to both files
- Count columns (R×N, E×N) using `utf8.RuneCountInString()` is acceptable — pure ASCII digits

## Implementation Notes

### dashboard_fileops.go changes:
1. Add `runewidth` import
2. Lines 55-74: replace `utf8.RuneCountInString()` for maxRWidth/maxEWidth with `runewidth.StringWidth()` (these display file counts which could be wide in theory, but per convention §7 exception for ASCII numbers this is acceptable — however for consistency with CJK paths, use runewidth)
3. Line 105: `truncatePath(path, pathWidth)` → `truncatePathBySegment(path, pathWidth)`
4. Line 106: `len(displayPath)` → `runewidth.StringWidth(displayPath)` for padding

### dashboard.go changes:
1. Add `runewidth` import
2. Line 418: replace `len(step.ToolName)` with `runewidth.StringWidth(step.ToolName)` for peak step
3. Lines 452-469: replace `len(name)` with `runewidth.StringWidth(name)` in labelWidth computation
4. Line 476: `truncateName` closure uses `truncRunes()` from truncate.go
