# 1.1: Create shared truncation utilities (truncate.go)

## Description

Create `internal/model/truncate.go` with 4 shared truncation/wrapping functions. These replace the 5+ inconsistent truncation implementations scattered across `subagent_overlay.go`, `dashboard_hook_panel.go`, `dashboard.go`, and `dashboard_fileops.go`.

## Reference Files

- `design/tech-design.md` Interface 1: Shared Truncation Utilities
- `docs/conventions/tui-dynamic-content.md` §3 (runewidth), §4 (path truncation), §5 (clamp), §6 (right-pad)
- Existing correct implementations:
  - `internal/model/detail.go:955` — `truncateFilePath` (segment-based, runewidth-aware)
  - `internal/model/calltree.go:1030` — `truncateLineToWidth` (lipgloss.Width-based)
  - `internal/model/subagent_overlay.go:356` — `truncRunes` (runewidth per-rune)

## Acceptance Criteria

- [ ] `truncatePathBySegment(path string, maxDisplayWidth int) string` — drops whole path segments from left, uses `runewidth.StringWidth`, preserves filename
- [ ] `truncateLineToWidth(line string, maxWidth int) string` — uses `lipgloss.Width()` for ANSI-aware truncation
- [ ] `truncRunes(s string, maxW int) string` — display-width-aware truncation with "…" suffix
- [ ] `wrapText(s string, maxDisplayWidth int) []string` — wraps at display-width boundaries using `runewidth`
- [ ] All edge case contracts from tech-design pass (empty string, zero width, single segment overflow)
- [ ] Unit tests cover all edge cases from Interface 1 Edge Case Contracts table (12 rows)
- [ ] `go test ./internal/model/ -run TestTruncate -v` passes

## User Stories

- **Story 5: Read Hook Statistics Without Text Overflow** — `truncatePathBySegment` provides the segment-based truncation format (`.../parent/file.go`)
  - Given a file path longer than display width → truncation drops whole segments from left
  - Given a CJK file path → truncation preserves complete UTF-8 characters and segments
  - Given a file path with no slashes → shows `...file.go` with leading ellipsis
  - Given a single-segment path longer than width → truncates from left with `...` prefix
  - Given an empty file path → returns empty string without crash

## Hard Rules

- Width calculation MUST use `runewidth.StringWidth()` for path/label content, `lipgloss.Width()` for ANSI strings
- `len()` MUST NOT be used for any visible width calculation
- `truncatePathBySegment` algorithm: split by `/`, drop from left, prepend `"..."`
- All functions must handle zero/negative maxDisplayWidth by returning `""`
