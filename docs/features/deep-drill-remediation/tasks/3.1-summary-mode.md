# 3.1: Summary mode for >50 sub-sessions

## Description

When a turn has more than 50 sub-sessions, replace the full sub-session list in the Call Tree inline expand with a single summary line showing count, average wall-time, and average tool calls per session.

## Reference Files

- `design/tech-design.md` Integration 8 (Summary Mode → Call Tree)
- `internal/model/calltree.go` — SubAgent expand rendering
- `internal/parser/types.go` — SubAgentStats for computing averages

## Acceptance Criteria

- [ ] Sub-session count >50: single summary line `"N sub-sessions (avg X.Xs, Y tools/session)"`
- [ ] Sub-session count <=50: full individual list (unchanged behavior)
- [ ] Summary values computed from actual data (count, avg duration, avg tools)
- [ ] Zero duration + zero tools: `"60 sub-sessions (avg 0.0s, 0 tools/session)"` — no division error
- [ ] Summary line truncated with `...` suffix if exceeds panel width at 80 columns
- [ ] Golden test: 52 sub-sessions → summary line renders within 80x24 panel width
- [ ] Golden test: 50 sub-sessions → full list renders (no summary)
- [ ] Golden test: 51 sub-sessions → summary line displays
- [ ] `go test ./internal/model/ -run TestCallTree -v` passes

## User Stories

- **Story 8: Get Summary for Sessions with Many SubAgents**
  - Given >50 sub-sessions → summary line with computed averages
  - Given exactly 50 → full list (threshold not exceeded)
  - Given 49 → full list
  - Given 51 → summary mode triggered
  - Given 60 with zero duration/tools → no division error
  - Given 1000 → summary line truncates to fit 80 columns

## Hard Rules

- Threshold is strictly `>50` (51 triggers, 50 does not)
- Summary line width uses `runewidth.StringWidth()` for truncation check
- Averages are computed from actual SubAgentStats data, not estimated

## Implementation Notes

Implementation in call_tree.go's SubAgent expand rendering:
1. Count sub-sessions (len of Children or equivalent)
2. If count > 50: compute averages from stats, format summary line
3. If count <= 50: render existing full list (no change)
4. Apply `truncateLineToWidth(summaryLine, contentWidth)` for safety

Average computation:
```go
totalDur := 0.0
totalTools := 0
for _, sub := range subs {
    totalDur += sub.Duration.Seconds()
    totalTools += sub.ToolCount
}
avgDur := totalDur / float64(len(subs))
avgTools := float64(totalTools) / float64(len(subs))
```
