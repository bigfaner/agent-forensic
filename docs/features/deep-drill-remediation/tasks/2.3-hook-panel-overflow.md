# 2.3: Fix hook panel overflow + CJK wrapping

## Description

Fix three width bugs in `dashboard_hook_panel.go`: (1) `renderHookStatsSection` ignores its `width` parameter, (2) `truncateStr` uses rune count instead of display width, (3) `wrapText` wraps at rune count instead of display width.

## Reference Files

- `design/tech-design.md` Integration 4 (Width-Safe Hook Panel)
- `internal/model/dashboard_hook_panel.go` тАФ `renderHookStatsSection` (line 75), `truncateStr` (line 268), `wrapText` (line 277)
- `internal/model/truncate.go` тАФ shared `truncRunes`, `wrapText` (from task 1.1)

## Acceptance Criteria

- [ ] `renderHookStatsSection` signature updated: `_ int` тЖТ `width int`, used for label truncation
- [ ] Hook labels truncate with `truncRunes()` at panel boundary, never extending past border
- [ ] `renderHookMarkerSelected` also uses `width` parameter for truncation
- [ ] Local `truncateStr` and `wrapText` functions removed тАФ uses shared versions from truncate.go
- [ ] `grep -n 'func truncateStr\|func wrapText' internal/model/dashboard_hook_panel.go` returns no matches
- [ ] Golden test with hook label >30 chars: truncation visible, no overflow
- [ ] Golden test with CJK hook description: wraps at display-width boundary, not rune count

## User Stories

- **Story 4: Read Hook Statistics Without Text Overflow** тАФ all hook labels truncate at panel border
  - Given long `HookType::Target` labels тЖТ truncate with `...` suffix, never past border
  - Given CJK hook description тЖТ wraps at correct column
  - Given zero hook entries тЖТ empty state, no crash

## Hard Rules

- `renderHookStatsSection` MUST use its `width` parameter тАФ no `_ int`
- Remove local `truncateStr` and `wrapText` тАФ use shared versions from truncate.go
- Add `runewidth` import
