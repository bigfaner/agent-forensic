# agent-forensic 使用指南

> 用 lazygit 的方式浏览你的 Claude Code 会话历史 — 一次按键即可看清 AI 做了什么、花了多久、有没有越界。

agent-forensic 是一个终端交互工具（TUI），用于回放和审查 Claude Code 的每次会话。你不再需要手动翻阅几千行的 JSONL 日志——所有信息都被组织成可视化的调用树，异常操作自动高亮标注。

**它能帮你：**
- 回顾某次会话里 AI 到底做了哪些操作
- 快速定位耗时异常或越权访问
- 实时监控正在运行的会话
- 统计各工具的使用频率和时间分布

---

## Quick Start

### 安装

从 [Releases](../../releases) 页面下载对应平台的二进制文件，或从源码构建：

```bash
git clone <repo-url> && cd agent-forensic
just build        # 需要 Go 1.26+
```

构建后当前目录生成 `agent-forensic` 可执行文件。

### 启动

```bash
./agent-forensic          # 中文界面（默认）
./agent-forensic -l en    # 英文界面
```

工具会自动扫描 `~/.claude/projects/` 下的所有会话记录（由 Claude Code 自动生成）。

### 第一次使用

启动后你会看到三栏界面：

```
┌─ Sessions ──────┐  ┌─ Call Tree ──────────────────────────────┐
│ ▸ 2026-05-12 .. │  │ Turn 1  用户: "Fix the bug"              │
│   2026-05-11 .. │  │   ├─ Read     src/main.go          0.2s │
│   2026-05-11 .. │  │   ├─ Bash     go test ./...        1.3s │
│   ...           │  │   └─ Write    src/main.go          0.1s │
│                 │  │ Turn 2  用户: "run again"                 │
│                 │  │   └─ Bash     go test ./...        0.8s │
├─────────────────┤  ├─────────────────────────────────────────┤
│                 │  │ Detail                                    │
│                 │  │ 选中任意节点，这里显示详细内容              │
└─────────────────┘  └─────────────────────────────────────────┘
```

**操作流程：**

1. 左侧选择一个会话（`↑`/`↓` 移动，`Enter` 选中）
2. 右上查看该会话的完整调用树
3. 选中某个工具调用，右下查看参数和输出
4. 按 `d` 一键查看所有异常，按 `s` 查看统计数据

**记住这 5 个键就够了：**

| 按键 | 作用 |
|------|------|
| `↑`/`↓` | 上下移动 |
| `Enter` | 确认选择 / 展开 |
| `Tab` | 切换面板焦点 |
| `d` | 查看异常诊断 |
| `q` | 退出 |

## 详细指南

### 界面总览

工具由三个面板组成，通过 `Tab` 键切换焦点：

| 面板 | 位置 | 用途 |
|------|------|------|
| **Sessions** | 左侧 | 所有会话列表，支持按日期/文件名搜索 |
| **Call Tree** | 右上 | 当前会话的操作记录，按对话轮次分组 |
| **Detail** | 右下 | 选中操作的具体参数、输出和 AI 思考过程 |

底部状态栏显示当前可用的快捷键提示、界面语言和版本号。

---

### 会话列表

左侧面板列出你的 Claude Code 会话记录。启动时显示最近 20 条，按 `G` 可以加载更多。

**搜索会话：** 按 `/` 进入搜索，支持以下方式：
- 按日期查找：输入 `2026-05-12` 或 `05-12`
- 按文件名查找：输入文件名中包含的文字
- `Enter` 确认搜索，`Esc` 取消并恢复完整列表

---

### 调用树

选中一个会话后，右上方展示它的完整操作历史。一次对话被组织成多个 **Turn（轮次）**，每个 Turn 包含用户的指令和 AI 执行的所有工具调用：

```
Turn 1  用户: "Fix the bug in auth"
  ├─ Read       src/auth.go           0.3s
  ├─ Bash       go test ./auth        2.1s
  └─ Write      src/auth.go           0.1s
Turn 2  用户: "commit this"
  └─ Bash       git commit -m "..."   0.5s
```

**操作方式：**

| 按键 | 说明 |
|------|------|
| `↑`/`↓` | 上下移动光标 |
| `Enter` | 展开或折叠一个 Turn |
| `n` | 直接跳到下一个 Turn |
| `p` | 直接跳到上一个 Turn |

**选中 Turn 节点时**，Detail 面板会展示用户发送的完整指令和本轮的工具使用统计。

**异常标记：**
- <span style="color:yellow">黄色</span> — 慢调用（超过 30 秒）
- <span style="color:red">红色</span> — 越权访问（读写项目目录外的文件）

---

### 详情面板

调用树中选中任何一个工具操作，右下方会显示它的完整信息：

- **工具参数** — 比如读取了哪个文件、执行了什么命令
- **执行结果** — 工具的输出内容，可滚动查看
- **AI 思考过程** — AI 在执行前的推理片段
- **敏感数据自动脱敏** — 包含 API_KEY、SECRET、TOKEN、PASSWORD 等关键词的值会被替换为 `***`

| 按键 | 说明 |
|------|------|
| `↑`/`↓` | 上下滚动内容 |
| `Enter` | 在完整/截断视图间切换 |

---

### 诊断报告

按 `d` 打开当前会话的诊断报告，它会汇总所有检测到的异常：

```
┌─ 诊断报告 ──────────────────────────────────┐
│                                              │
│ ⚠ 慢调用 #1                                  │
│   Bash "go build ./..." 耗时 45.2s           │
│   位置: Turn 3, Step 2                       │
│   ── AI 思考 ──                              │
│   "This build is taking unusually long..."   │
│                                              │
│ 越权访问 #1                                   │
│   Write "/etc/config.yaml" (项目目录外)       │
│   位置: Turn 5, Step 1                       │
│                                              │
└──────────────────────────────────────────────┘
```

每条异常包含具体操作内容、发生位置和 AI 当时的思考。用方向键切换条目，按 `Enter` 可直接跳转到调用树的对应位置查看上下文。

按 `Esc` 或 `q` 关闭诊断报告。

---

### 统计面板

按 `s` 打开统计仪表盘，了解一次会话的全貌：

- **工具调用次数** — 柱状图展示各工具使用频率
- **耗时分布** — 百分比条展示各工具占用的总时间
- **峰值步骤** — 最慢的那一次操作
- **总耗时** — 会话从开始到结束的总时间
- **自定义工具** — 细分 Skill、MCP 工具、Hook 的调用情况

| 按键 | 说明 |
|------|------|
| `1` | 打开会话选择器，切换到其他会话的统计 |
| `r` | 刷新统计数据 |
| `s` 或 `Esc` | 关闭面板 |

---

### 实时监控

在调用树面板中按 `m` 可以开启实时监控模式。当你有一个正在运行的 Claude Code 会话时，新的操作会实时出现在调用树中，带有青色 `[NEW]` 标记闪烁提示（3 秒后自动消失）。

适用于以下场景：
- 让 Claude Code 在另一个终端窗口工作，同时在这里实时观察它的行为
- 监控长时间运行的自动化任务

再按一次 `m` 关闭监控。

---

### 语言切换

支持中文和英文两种界面语言：

- 启动时通过 `-l en` 或 `-l zh` 指定
- 运行中随时按 `L` 键即时切换

### 常见问题

**启动后左侧面板为空？**

确保你已经用过 Claude Code，它会在 `~/.claude/projects/` 下自动生成会话记录。如果目录不存在或没有 `.jsonl` 文件，说明还没有可查看的会话。

**界面显示错乱？**

终端窗口需要至少 80 列 x 24 行。过小的终端会显示警告。

**某个会话显示为损坏？**

如果会话文件中超过一半的内容无法解析（可能是文件损坏或不完整），工具会标记为损坏会话并跳过。

### 典型使用场景

**场景 1：回顾昨天的工作**

按 `/` 输入昨天的日期 → 从结果中选择会话 → 浏览调用树了解 AI 做了什么。

**场景 2：排查一次耗时很长的会话**

选择会话 → 按 `s` 查看统计面板，找到耗时最长的操作 → 按 `d` 查看是否被标记为慢调用异常。

**场景 3：审查安全性**

选择会话 → 按 `d` 打开诊断报告 → 检查是否有越权文件访问的红色标记。

**场景 4：实时跟踪正在运行的任务**

选择当前活跃会话 → 按 `m` 开启监控 → 在另一个终端观察 AI 的实时操作。
